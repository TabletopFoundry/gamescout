# Your First Recommendations

By the end of this page you'll have a working taste profile and you'll understand exactly why each game appeared on your Discover feed.

## 1. Take the quiz

Open **[http://localhost:3000/quiz](http://localhost:3000/quiz)**.

The quiz has two parts:

1. **Rate ten well-known games.** Pick one of `loved`, `liked`, `neutral`, `disliked`, or `haven't played` for each. These ratings drive most of your profile — the games you love define the category and mechanic weights.
2. **Answer preference questions.** Player count, complexity tolerance, and ideal playtime. These produce hard-ish constraints used by the scoring engine.

Quiz answers are written through `POST /api/quiz` as a single bulk upsert. You can retake the quiz at any time; old answers for the same game are replaced.

## 2. Read your recommendations

Open **`/discover`**.

The page makes a single call to `GET /api/recommendations` and renders up to 24 games. Each card shows:

- The game's name, image, and BGG rating
- A computed **score** (higher = better fit)
- A human-readable **reason** like *"You loved 3 strategy games with worker placement"*

The reason isn't generated by an LLM — it's produced by the same scoring code that ranks the games. If it says "complexity match", that's because the complexity-distance bonus dominated. See [Recommendation Engine](../concepts/recommendation-engine) for the formula.

## 3. Refine via mood

Scroll down on `/discover` to the **Browse by Mood** section. Each mood is a hard filter applied server-side in SQL:

- **Quick Party Game** — 4+ players, complexity ≤ 2, ≤ 45 minutes
- **Deep Strategy** — complexity ≥ 3.5, ≥ 90 minutes
- **Cozy Two-Player** — max 2 players, complexity ≤ 2.5
- *(…and five more)*

Moods are independent of your taste profile — they're a quick way to slice the catalog when you already know what you're in the mood for.

## 4. Inspect a game

Click any card. The detail page shows:

- Full description, complexity tier, designer, publisher, BGG rank
- Current retailer prices and any active deals
- Reviews and average rating
- "You may also like" — five similar games computed from category/mechanic overlap
- A **Log a Play** form to record a session

## 5. See the loop close

Once you've added a game to your collection (`POST /api/collection`) or logged a play (`POST /api/play-logs`), the recommendation engine has new signal. Refresh `/discover` and watch the scores shift.
